git使用前配置

在使用git钱，需要告诉git你是谁，在向git仓库中提交时需要用到。

  • 配置提交人姓名：git config --global user.name 提交人姓名
  • 配置提交人邮箱：git config --global user.email 提交人邮箱
  • 查看git配置信息：git config --list

如果需要修改以上信息，只需要再次执行配置命令即可

配置命令只需执行一次。之后切换仓库也是这个信息。

git提交步骤

  • git init 初始化git仓库
  • git status 查看文件状态
  • git add 文件 添加文件进入追踪状态
  • git commit -m 提交信息 向仓库提交版本
  • git log 查看提交记录

git撤销

用暂存区文件覆盖工作目录中的文件：git checkout 文件名

将文件从暂存区中删除：git rm --cached 文件名

将git仓库中指定的更新记录恢复过来，并且覆盖暂存区和工作目录：git reset --hard 提交id

分支命令

  • git branch 查看分支
  • git branch 分支名称 创建分支
  • git checkout 分支名称 切换分支(切换时一定要注意提交文件，如果暂存区有文件没提交那这个文件就会出现在切换后的分支中，造成文件混乱)
  • git merge 来源分支 合并分支
  • git branch -d 分支名称 删除分支（只有合并分支后才允许删除，-D强制删除）

暂时保存分支

在git中，可以暂时提取分支上所有的改动并存储，让开发人员得到一个干净的工作副本，从而临时转向其他副本

使用场景：分支临时切换

临时存储改动：git stash

恢复改动：git stash pop
