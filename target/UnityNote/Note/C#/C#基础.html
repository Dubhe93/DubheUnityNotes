引用类型

面向对象中new出来的对象都是引用类型

C#默认自带的引用类型有object（所有类的父类）、dynamic（动态数据类型，可以存储任何值）、string（字符型）


指针类型

指针类型变量存储另一种类型的内存地址。C# 中的指针与 C 或 C++ 中的指针有相同的功能。

例如：int* iptr;


交换变量值

int a=1;int b=2;

方法1：

int c=a;

a=b;

b=c;

这种方法简洁运行速度快，但需要开辟空间，内存占用大（项目中常用这种方法）

方法2：

a=a+b;

b=a-b

a=a-b

这种方法需要三次计算速度慢，但内存占用小（只有项目有特殊要求时可能才会使用）

参数数组

当我们创建一个方法，在不清楚参数个数的时候，可以将参数设置成数组来解决

但每次传递参数都需要重新创建一个数组，就很麻烦，这时就可以使用参数数组

params

被params修饰的形参必须是一个数组，而且位于形参的最后一位，之后调用方法就可以不用创建数组，只需要传输对应的参数即可


常量

C#中的常量通过const修饰，一旦修饰直至程序结束运行，其值不能修改。

使用 const 关键字来声明某个常量字段或常量局部变量。 常量字段和常量局部变量不是变量并且不能修改。 常量可以为数字、布尔值、字符串或 null 引用。


枚举

枚举类型 是由基础整型数值类型的一组命名常量定义的值类型。 若要定义枚举类型，请使用 enum 关键字并指定枚举成员 的名称

默认情况下，枚举成员的关联常数值为类型 int；它们从零开始，并按定义文本顺序递增 1。 可以显式指定任何其他整数数值类型作为枚举类型的基础类型。

枚举的底层用的是整型，枚举的成员名称相当于变量名，在不赋值的情况下默认从第一个成员开始赋值，从0开始。

这么做的好处是数据通过整型存储体积小，相互对比判断速度快，而展示给用户时展示成员名，规范了命令也能更好的理解

结构体内部可以写函数


委托

委托方法在项目中经常使用

使用委托时需要先进行声明，声明方法和定义函数类似但不需要函数体

在实际开发中我们每个人都是负责一个模块，如果我们都在开发中，都没有做完，也不知道对方怎么写的，但负责实现玩家死亡的模块的人需要调用死亡UI

而负责开发ui的人还没有写死亡UI的的函数，这时就可以使用委托。在人物死亡的函数中留下一个委托，之后调用人物死亡函数的开发者只需要调用时将死亡ui的函数名传输进入死亡函数就行了

委托数组写法

Action委托和Func委托

这两个委托是C#为我们准备的委托

一些简单的委托可以通过它们直接实现，为我们节省时间

Action委托可以指向无返回值的方法

Func委托和Action委托不同，Func必须带有返回值

Func的返回值必须是最后一位，前几位可以作为参数，如果没有参数就不写，但返回值一定要。
